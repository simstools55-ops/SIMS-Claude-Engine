# Engine01 Audit Report

## Source reviewed

Prototype file:

```text
SIMSv2/engine01.md
```

## Prototype role

Engine01 is defined as **Project Intake Engine**.

Its purpose is to organize, validate, and normalize SEO improvement input data so that all downstream engines can work from a common format.

## Evaluation

### Keep

The core responsibility is correct and should be preserved.

Engine01 should not perform SEO analysis. This separation is important because it prevents later engines from receiving unstable or partially interpreted input.

### Modify

The input model must be updated from manual input to AI Exchange input.

Old input:

- Main keyword
- Article URL
- Search Console top 10 queries
- Optional article body
- Blog index

New input:

- Improvement_Request generated by SIMS
- Article URL
- Main query candidate
- Primary sub query candidates
- Supporting query table, recommended max 20
- H1 / title tag / meta description when available
- Optional article body
- Optional Blog Profile / Dynamic Knowledge

### Do not change

Engine01 must continue to avoid the following:

- Search intent analysis
- SERP analysis
- Article rewriting
- Title generation
- FAQ generation
- Internal link recommendations
- Publishing decisions

Those belong to later engines.

## Final decision

Engine01 remains **Project Intake Engine**.

Search Intent will be handled by a later engine, likely Engine02 or a renamed `Search Intent Engine`, after the intake contract is stable.
