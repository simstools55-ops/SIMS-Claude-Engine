# Engine01 AI Exchange Intake Contract

## Engine

```text
Engine ID: E01
Engine Name: Project Intake Engine
Role: Input validation and normalization
```

## Primary input

Engine01 receives one `Improvement_Request` generated by SIMS.

## Required fields

- Request ID
- Article URL
- Main query candidate
- Search Console snapshot
- At least one query row with impressions, clicks, CTR, and position

## Recommended fields

- Primary sub queries, 3 to 5
- Supporting queries, up to 15
- Total query table size, max 20
- Current H1
- Current title tag
- Current meta description
- Current smartphone description, if available
- Article category
- Article tags
- Current article body, pasted manually if not fetched by SIMS

## Optional fields

- Blog Profile
- Dynamic Knowledge
- Improvement history
- Internal link candidates
- User notes

## Missing data handling

Engine01 must not invent missing fields.

If a field is missing, Engine01 must mark it as one of:

```text
Available
Missing
Optional Missing
User Required
```

## Normalized output

Engine01 outputs a normalized intake object for downstream engines.

```text
Normalized_Intake
├── Request_Metadata
├── Article_Profile
├── Search_Console_Snapshot
├── SEO_Metadata_Current
├── Query_Set
├── Knowledge_Context
├── Missing_Data
└── Ready_For_Next_Engine
```

## Gate decision

Engine01 returns one of three states:

```text
PASS
PASS_WITH_WARNINGS
STOP_USER_INPUT_REQUIRED
```

## Downstream contract

Only if the gate decision is `PASS` or `PASS_WITH_WARNINGS`, the next engine may proceed.
