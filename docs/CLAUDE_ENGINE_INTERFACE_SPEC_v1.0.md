# Claude Engine Interface Specification v1.0

## 1. Purpose

This document defines how Claude Engine handles SIMS AI Exchange v1.0.

Claude Engine is not the owner of the exchange specification. It is a compatible implementation.

## 2. Input

Claude Engine must accept `Improvement_Request.md` generated by SIMS.

Input includes:

- URL
- Current SEO metadata
- Search Console snapshot
- Main query candidate
- Sub query candidates
- Supporting queries
- Improvement goal
- User-provided article body when available

## 3. Output

Claude Engine must produce `Improvement_Result.md` with:

- Final main query
- Final sub queries
- Recommended H1
- Recommended title tag
- Recommended meta description
- Smartphone meta description
- Article changes
- Beginner explanations
- WordPress reflection guide
- Completion checklist
- Reasoning summary
- Next action

## 4. Engine Responsibilities

Claude Engine is responsible for:

- Search intent analysis
- Query interpretation
- SEO metadata generation
- Article structure improvement
- FAQ additions
- Internal link suggestions
- Beginner-friendly explanation

Claude Engine is not responsible for:

- Search Console API connection
- Spreadsheet management
- KPI history storage
- Measurement calculation
- WordPress publishing

## 5. Quality Rules

- Never return only a generic improvement summary.
- Always include actionable copy/paste outputs.
- Always distinguish H1, title tag, and meta description.
- Always explain where the user should paste each item.
- Do not expose hidden reasoning; provide a concise reasoning summary only.

## 6. Compatibility

Compatible with SIMS AI Exchange Standard v1.0.
